У сервиса есть два варианта запуска:

Первый: обычный
Через каждый промежуток времени, указанный в переменной окружения CHECK_GAP_SECONDS сервис делает запрос на метеостанцию
и записывает данные от неё в таблицу weather_data_raw, если запрос сделать не получилось (станция часто выдаёт 404),
то время, за которое сервис не смог получить данные отправляется в "бэк-лог". Данные о времени из "бэк-лога"
сервис получает каждый промежуток времени, указанный в переменной окужения CHECK_FAILED_TIME_GAP_SECONDS и чистит
"бэк-лог"

Второй: ручной
Сервис можно запустить в ручном режиме указав переменны окружения TIME_FROM и TIME_TO в формате YYYY-MM-DD hh:mm, сервис
саберёт данные за это время и сохранит в бд, после чего закончит работу
